# Plan: 용도지역 프리셋 & 수익성 간이 분석

> CPS Framework | 2026-04-03 | Phase: Plan
> Design: designs/002-zoning-presets-profitability.md

---

## Context (왜 필요한가)

### 현재 상태
- 용적률 시뮬레이터 MVP가 운영 중 (대지면적·용적률·건폐율·층수 → 3D 시각화)
- 사용자가 슬라이더를 자유롭게 조작하지만, **현실적으로 어떤 값이 맞는지 모름**
- 계산 결과가 m²/층수로만 나오며, **"그래서 이 건물이 돈이 되는가?"** 에 답하지 못함

### 배경
- 부동산 초보자의 가장 큰 질문: "이 땅에 뭘 지을 수 있고, 수익이 나는가?"
- 용도지역(주거/상업/공업 등)에 따라 용적률·건폐율 법적 상한이 **고정**되어 있음
- 현재 시뮬레이터는 이 법적 제약을 반영하지 않아 비현실적인 값 입력 가능

### 대상 사용자
- 부동산 투자/개발에 관심 있는 초보자
- "이 땅을 사면 어떤 건물을 짓고 얼마나 벌 수 있을까?" 를 알고 싶은 사람

---

## Problem (무엇이 문제인가)

### P1. 현실감 부재 — 용도지역 제약 미반영
- 사용자가 용적률 1500%, 건폐율 80%로 설정해도 아무런 가이드 없음
- 실제로 제2종 일반주거지역은 용적률 250% 이하, 건폐율 60% 이하
- **초보자는 어떤 값이 현실적인지 판단 불가**

### P2. 실용성 부재 — "그래서 어쩌라고?"
- 건축면적·연면적·층수만으로는 의사결정 불가
- 사용자가 진짜 알고 싶은 것:
  - 이 건물 짓는 데 **대략 얼마** 드는가?
  - 임대하면 **월 수익**이 얼마나 나오는가?
  - **투자 대비 수익률**은?

### P3. 학습 연결 부재
- 용도지역 개념이 "개념 사전"에도 없음
- 시뮬레이터와 개념 학습 간 연결 고리가 약함

---

## Solution (어떻게 해결하는가)

### 기능 범위 (Scope)

#### S1. 용도지역 프리셋 시스템
사용자가 용도지역을 선택하면 용적률·건폐율 상한이 자동 설정되고, 슬라이더 범위가 제한됨.

**프리셋 목록** (서울시 기준, 국토계획법):

| 용도지역 | 건폐율 상한 | 용적률 상한 | 대표 예시 |
|----------|-----------|-----------|----------|
| 제1종 전용주거 | 50% | 100% | 단독주택지 |
| 제2종 전용주거 | 50% | 150% | 공동주택지 |
| 제1종 일반주거 | 60% | 200% | 저층 주거 |
| 제2종 일반주거 | 60% | 250% | 중층 주거 |
| 제3종 일반주거 | 50% | 300% | 고층 주거 |
| 준주거 | 70% | 500% | 주거+상업 혼합 |
| 일반상업 | 80% | 1000% | 상업 중심 |
| 근린상업 | 70% | 600% | 동네 상가 |
| 준공업 | 70% | 400% | 공업+주거 혼합 |
| 자유 입력 | - | - | 기존 방식 유지 |

**동작 방식**:
1. 시뮬레이터 상단에 용도지역 선택 드롭다운 추가
2. 선택 시 → 용적률·건폐율 슬라이더의 max 값이 해당 상한으로 변경
3. 현재 값이 상한 초과 시 → 자동으로 상한까지 조정
4. "자유 입력" 선택 시 → 기존처럼 제한 없이 사용
5. 선택된 용도지역 설명이 인풋 패널 상단에 한 줄로 표시

#### S2. 수익성 간이 분석 패널
시뮬레이터 결과에 **간이 수익 분석** 섹션을 추가.

**표시 항목**:

| 항목 | 계산식 | 비고 |
|------|--------|------|
| 예상 건축비 | 연면적 × 평당 건축비 | 용도별 단가 차등 |
| 예상 월 임대수익 | 연면적 × 평당 월임대료 | 용도별 단가 차등 |
| 연 임대수익 | 월 임대수익 × 12 | |
| 단순 투자회수 기간 | 건축비 ÷ 연 임대수익 | 토지비 미포함 |
| 표면 수익률 | 연 임대수익 ÷ 건축비 × 100 | % |

**단가 기준** (서울 평균, 사용자 조정 가능):

| 용도 | 평당 건축비 | 평당 월임대료 |
|------|-----------|-------------|
| 주거 | 700만원 | 5만원 |
| 상업 | 800만원 | 8만원 |
| 공업 | 500만원 | 3만원 |

**동작 방식**:
1. ScenarioInterpretation 하단에 "수익성 간이 분석" 섹션 추가
2. 용도지역 선택에 따라 단가가 자동 적용
3. 단가를 직접 수정할 수 있는 작은 입력 필드 제공
4. "이 분석은 참고용이며, 실제 투자 판단의 근거로 사용할 수 없습니다" 면책 표시

#### S3. 개념 사전 확장
- **용도지역** 개념 추가 (category: "regulation")
- **건축비** 개념 추가 (category: "finance")
- **임대수익률** 개념 추가 (category: "finance")

### 트레이드오프

| 판단 | 선택 | 이유 |
|------|------|------|
| 단가 데이터 소스 | 하드코딩 (사용자 수정 가능) | 외부 API 의존 없이 SSG 유지. 사용자가 직접 조정 가능하므로 정확성 보완 |

| 용도지역 데이터 범위 | 서울시 기준 9개 | 전국 데이터는 과도. 대표적 9개면 학습용으로 충분 |
| 수익 계산 정밀도 | 초간이 (5개 항목) | 정밀 분석은 전문 도구 영역. 여기선 "감 잡기"가 목적 |
