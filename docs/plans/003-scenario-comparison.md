# Plan: 시나리오 비교 & 실제 사례 프리셋

> CPS Framework | 2026-04-03 | Phase: Plan

---

## Context (왜 필요한가)

### 현재 상태
- 시뮬레이터에서 하나의 조건만 볼 수 있음
- 용도지역 프리셋과 수익성 분석이 추가되어 실용성은 올라감
- 하지만 "이 조건이 좋은지 나쁜지"를 판단하려면 **비교 대상**이 필요

### 배경
- 부동산 의사결정의 핵심은 **비교**: "A 땅 vs B 땅", "5층 vs 10층"
- 초보자는 숫자만 보고 좋고 나쁨을 판단하기 어려움
- 실제 서울 주요 지역의 사례가 있으면 "기준점"이 생김

### 대상 사용자
- "강남 100평 vs 마포 200평, 어디가 수익이 나을까?" 비교하고 싶은 사람
- 처음 시뮬레이터를 열었을 때 "뭘 넣어야 할지 모르겠는" 사람

---

## Problem (무엇이 문제인가)

### P1. 단일 시나리오의 한계
- 한 번에 하나의 조건만 볼 수 있어 비교 불가
- 값을 바꿔가며 기억에 의존해 비교해야 함

### P2. 초보자의 진입 장벽
- 시뮬레이터를 처음 열면 기본값(330m², 200%, 60%, 5층)만 보임
- "실제로 어떤 값이 현실적인지" 감을 잡기 어려움
- 실제 사례가 있으면 바로 체험 가능

### P3. 학습 효과 극대화
- "같은 땅에 주거 vs 상업" 같은 비교가 가장 교육적
- 현재는 사용자가 직접 이를 시도해야 함

---

## Solution (어떻게 해결하는가)

### S1. 시나리오 저장 & 비교 (2개)

**핵심 동작**:
1. 현재 시뮬레이터 상태를 "시나리오 A"로 저장 (버튼 클릭)
2. 값을 변경한 후 "시나리오 B"와 나란히 비교
3. 최대 2개까지 비교 (A vs B)
4. 주요 지표를 하이라이트: 어느 쪽이 수익률 높은지, 건축비 적은지

**비교 항목** (하이라이트 표시):
- 용도지역
- 연면적
- 예상 건축비
- 월 임대수익
- 표면 수익률
- 투자회수 기간

**UI**: 시뮬레이터 하단에 비교 카드 (접기/펼치기)

### S2. 실제 사례 프리셋

버튼 하나로 실제 서울 지역의 대표 조건을 로드.

**프리셋 목록** (6개):

| 프리셋 | 대지면적 | 용도지역 | 용적률 | 건폐율 | 층수 | 설명 |
|--------|---------|---------|--------|--------|------|------|
| 강남 단독주택지 | 330m² | 제2종 일반주거 | 250% | 60% | 4 | 강남 대치동 100평 단독 |
| 마포 다세대 | 200m² | 제2종 일반주거 | 250% | 60% | 5 | 마포구 60평 다세대 |
| 홍대 근생 | 150m² | 근린상업 | 600% | 70% | 8 | 홍대입구역 상가건물 |
| 강남역 오피스 | 500m² | 일반상업 | 1000% | 80% | 15 | 강남역 대로변 오피스 |
| 성수 카페거리 | 250m² | 준공업 | 400% | 70% | 5 | 성수동 카페/공방 |
| 은평 신축빌라 | 280m² | 제1종 일반주거 | 200% | 60% | 4 | 은평구 소규모 빌라 |

**동작 방식**:
1. 시뮬레이터 상단에 "실제 사례" 버튼 그룹
2. 클릭 시 해당 프리셋의 모든 값(용도지역 + 파라미터)을 한 번에 로드
3. 사용자가 이후 자유롭게 수정 가능

**변경 파일 수**: 3개 수정 + 2개 신규 = **5개** → 복잡도: **보통**

### 트레이드오프

| 판단 | 선택 | 이유 |
|------|------|------|
| 비교 개수 | 최대 2개 | 3개 이상은 UI 복잡. A vs B가 가장 직관적 |
| 저장 방식 | 메모리 (새로고침 시 초기화) | localStorage는 MVP에 과도. 추후 추가 가능 |
| 프리셋 수 | 6개 | 서울 주요 지역 커버. 너무 많으면 선택 피로 |
| 단가 동기화 | 프리셋 로드 시 용도지역 기본 단가로 리셋 | 사용자 커스텀 단가는 유지 안 함 (직관적 동작) |

### 제외 항목
- 3개 이상 시나리오 비교
- 시나리오 영구 저장 (localStorage/DB)
- 사용자 정의 프리셋 만들기
- URL 공유 (쿼리스트링 기반)
- 지도 기반 위치 선택
